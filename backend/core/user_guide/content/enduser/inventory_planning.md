# Inventory Planning

## Purpose

Inventory Planning shows item-level demand and ordering progress before distribution and procurement decisions.

## Main numbers

- Needed is the total required quantity from submitted applications.
- Ordered is the quantity already moved through submitted ordering or fund request actions.
- Pending is the quantity still not covered.
- Excess appears when ordered quantity is higher than needed.

## How to use

- Use metric cards to understand Article, Aid, and Project totals.
- Use filters for item type, application status, combo, order status, balance, and search.
- Expand a row to see beneficiary-level demand.
- Use exports after filtering if a focused planning sheet is needed.

## Important logic

- District, Public, Institution, and Others are all included in planning.
- Expanded views separate beneficiary types where required.
- Partial progress is allowed. A row can be partly ordered and still show pending balance.
- If the source looks wrong, correct Application Entry or Order & Fund Request first.
